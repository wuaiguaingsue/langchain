# 文本嵌入模型

:::info
前往 [集成](/docs/integrations/text_embedding/) 查看内置文本嵌入模型提供商的文档。
:::

`Embeddings` 类是一个用于与文本嵌入模型交互的类。有许多嵌入模型提供商（如 OpenAI、Cohere、Hugging Face 等）——此类旨在为所有这些提供商提供一个标准接口。

嵌入将文本表示为向量。这很有用，因为它意味着我们可以在向量空间中处理文本，并执行诸如语义搜索之类的操作，在语义搜索中我们寻找在向量空间中最相似的文本。

LangChain 中的基础 `Embeddings` 类提供了两种方法：一种用于嵌入文档，另一种用于嵌入查询。前者 `.embed_documents` 接收多个文本作为输入，而后者 `.embed_query` 接收单个文本。将这两种方法分开的原因是某些嵌入提供商对文档（用于搜索）和查询（搜索本身）有不同的嵌入方法。
`.embed_query` 将返回一个浮点数列表，而 `.embed_documents` 返回一个浮点数列表的列表。

## 开始使用

### 设置

import EmbeddingTabs from "@theme/EmbeddingTabs";

<EmbeddingTabs customVarName="embeddings_model" />

### `embed_documents`
#### 嵌入文本列表

使用 `.embed_documents` 嵌入字符串列表，返回嵌入列表：

```python
embeddings = embeddings_model.embed_documents(
    [
        "Hi there!",
        "Oh, hello!",
        "What's your name?",
        "My friends call me World",
        "Hello World!"
    ]
)
len(embeddings), len(embeddings[0])
```

```output
(5, 1536)
```


### `embed_query`
#### 嵌入单个查询
使用 `.embed_query` 嵌入单个文本（例如，用于与其他嵌入的文本片段进行比较）。

```python
embedded_query = embeddings_model.embed_query("What was the name mentioned in the conversation?")
embedded_query[:5]
```

```output
[0.0053587136790156364,
 -0.0004999046213924885,
 0.038883671164512634,
 -0.003001077566295862,
 -0.00900818221271038]
```
